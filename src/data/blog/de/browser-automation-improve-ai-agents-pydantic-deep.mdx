## Teil 1: BrowserCapability

### Installation

```bash
pip install 'pydantic-deep[browser]'
playwright install chromium
```

### Die 9 Tools

```python
from pydantic_deep.capabilities import BrowserCapability

agent = create_deep_agent(
    model="anthropic:claude-opus-4-6",
    extra_capabilities=[BrowserCapability(
        allowed_domains=["github.com", "docs.python.org"],
        auto_screenshot=True,
        headless=False,  # sichtbares Fenster standardmäßig
    )]
)
```

| Tool | Was es tut |
|------|-----------|
| `navigate(url)` | Navigiert zur URL, gibt Inhalt + optionalen Screenshot zurück |
| `click(selector)` | Klickt Element per CSS-Selektor |
| `type_text(selector, text)` | Tippt in Eingabefeld |
| `get_text(selector)` | Extrahiert Text aus Element |
| `screenshot()` | Erfasst aktuellen Seitenzustand |
| `scroll(direction, amount)` | Scrollt Seite hoch/runter |
| `go_back()` | Browser zurück |
| `go_forward()` | Browser vorwärts |
| `execute_js(script)` | Führt beliebiges JavaScript aus |

Jedes Tool gibt ein `BrowseResult` zurück: `url`, `title`, `content`, `screenshot`, `error`.

### Sicherheitsdesign

**Single-Tab.** Ein Tab auf einmal — bewusste Entscheidung, keine Einschränkung. Mehrere Tabs würden erfordern, dass der Agent Tab-Kontext verfolgt.

**Domain-Allowlist.** Der Agent kann nur zu explizit erlaubten Domains navigieren. Navigationsversuche woanders geben einen Fehler zurück.

**Automatische Popup-Abfangung.** Cookie-Banner, Login-Dialoge, Abonnement-Eingabeaufforderungen — automatisch abgefangen.

**Inhaltsabschneidung.** Seiteninhalt wird abgeschnitten, bevor er an das Modell übergeben wird.

### Browser-Lebenszyklus

```python
# Lebenszyklus über wrap_run-Muster:
# Chromium startet → Agent läuft → Chromium stoppt
# Garantiert bei: Erfolg / Ausnahme / Abbruch
```

**CLI-Flags:**

```bash
pydantic-deep tui --browser                      # Browser aktivieren (standardmäßig deaktiviert)
pydantic-deep tui --browser --browser-headless   # kein Fenster (für CI)
pydantic-deep tui --browser --browser-headed     # sichtbares Fenster (Standard)

# Kombiniert mit Docker Sandbox
pydantic-deep run "research competitors on GitHub" \
  --browser \
  --sandbox docker
```

## Teil 2: /improve — Die Selbstverbesserungsschleife

### Was /improve macht

Nach einer Agenten-Sitzung `/improve` in der TUI ausführen. Es analysiert die vollständige Sitzung und extrahiert zwei Kategorien von Erkenntnissen:

**`UserFactInsight`** — was der Agent über Sie und Ihre Präferenzen gelernt hat.

**`AgentLearningInsight`** — Muster und Strategien, die der Agent entdeckt hat.

Beide Typen schreiben in MEMORY.md, das beim Start jeder nachfolgenden Sitzung geladen wird.

### Die Meta-Harness-Erkenntnis: Rohe Traces >> Zusammenfassungen

Wir verglichen zwei Eingaben für den Syntheseschritt:
1. Für Menschen lesbare Zusammenfassungen des Geschehenen
2. Rohe Tool-Traces — die tatsächliche Sequenz von Tool-Aufrufen und Antworten

Rohe Traces schnitten deutlich besser ab. Zusammenfassungen verlieren die Fehler-/Wiederherstellungsmuster, die das eigentliche Lernsignal enthalten. `tool_log.jsonl` (pro Sitzung geschrieben) erfasst den vollständigen Tool-Trace. `/improve` liest dies direkt.
