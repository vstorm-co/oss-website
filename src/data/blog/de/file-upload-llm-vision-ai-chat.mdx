> **TL;DR:** Version 0.2.2 des Full-Stack AI Agent Templates fügt Datei-Upload mit LLM-Vision-Unterstützung hinzu. Bilder (PNG, JPG, WebP) werden als `BinaryContent` direkt an die LLM Vision API gesendet — das Modell sieht sie. PDFs werden serverseitig mit PyMuPDF geparst (Textblöcke + Tabellen in Markdown konvertiert). DOCX-Dateien werden mit python-docx geparst. Der gesamte geparste Inhalt wird in den Kontext des Agenten eingefügt. Der Upload-Flow nutzt REST für die Dateibehandlung (Validierung, Parsing, Speicherung) und WebSocket für Chat-Nachrichten (Referenzierung per File-IDs). Ein `ChatFile`-Datenbankmodell verfolgt Eigentümer, MIME-Typ, Speicherpfad und gecachten geparsten Inhalt. Der PDF-Parser ist über die Umgebungsvariable `CHAT_PDF_PARSER` konfigurierbar (PyMuPDF, LlamaParse oder LiteParse). Funktioniert mit allen 5 unterstützten AI-Frameworks.

## Die Architektur: Drei Pfade für drei Dateitypen

Wenn ein Benutzer eine Datei im Chat hochlädt, können je nach Dateityp drei Dinge passieren:

```
Benutzer lädt Datei hoch
  │
  ├── Bild (png/jpg/gif/webp)
  │     → Speichern in media/
  │     → Laden als BinaryContent
  │     → Senden an LLM Vision API
  │     → Agent "sieht" das Bild
  │
  ├── PDF
  │     → Speichern in media/
  │     → Parsen mit PyMuPDF (Text + Tabellen)
  │     → Geparsten Text in Agent-Prompt einfügen
  │     → Agent liest den Dokumentinhalt
  │
  └── DOCX
        → Speichern in media/
        → Parsen mit python-docx
        → Geparsten Text in Agent-Prompt einfügen
        → Agent liest den Dokumentinhalt
```

Die zentrale Designentscheidung: **Bilder und Dokumente nehmen unterschiedliche Wege**. Bilder benötigen multimodale LLM-Fähigkeiten (Vision API). Dokumente benötigen serverseitige Textextraktion. Beide landen im Kontext des Agenten, aber über grundlegend verschiedene Mechanismen.

## Schritt 1: Upload-Endpoint mit Validierung

```python
@router.post("/upload", response_model=FileUploadResponse,
             status_code=status.HTTP_201_CREATED)
async def upload_file(
    file: UploadFile = File(...),
    file_upload_svc: FileUploadSvc = None,
    current_user: CurrentUser = None,
) -> Any:
    data = await file.read()
    is_valid, error = file_upload_svc.validate_upload(
        file.content_type, len(data)
    )
    if not is_valid:
        raise HTTPException(status_code=400, detail=error)

    file_type = file_upload_svc.classify_file(
        file.content_type or "", file.filename or "unknown"
    )
    parsed_content = await file_upload_svc.parse_content(
        data, file_type, file.content_type or ""
    )

    storage = get_file_storage()
    storage_path = await storage.save(
        str(current_user.id), file.filename or "unknown", data
    )

    chat_file = await file_upload_svc.create_chat_file(
        user_id=current_user.id,
        filename=file.filename or "unknown",
        mime_type=file.content_type or "application/octet-stream",
        size=len(data),
        storage_path=storage_path,
        file_type=file_type,
        parsed_content=parsed_content,
    )
    return FileUploadResponse(
        id=chat_file.id, filename=chat_file.filename,
        mime_type=chat_file.mime_type, size=chat_file.size,
        file_type=chat_file.file_type,
    )
```

## Schritt 2: PDF-Parsing mit PyMuPDF

```python
@staticmethod
def _parse_pdf_content(data: bytes) -> str | None:
    import pymupdf
    doc = pymupdf.open(stream=data, filetype="pdf")
    texts = []
    for page in doc:
        blocks = page.get_text("blocks")
        for b in blocks:
            if b[6] == 0:
                text = b[4].strip()
                if text:
                    texts.append(text)
        tables = page.find_tables()
        if tables and tables.tables:
            for table in tables.tables:
                df = table.to_pandas()
                if not df.empty:
                    texts.append(df.to_markdown(index=False))
    doc.close()
    return "\n\n".join(texts) if texts else None
```

```python
from pydantic_ai.messages import BinaryContent

for fid in file_ids:
    chat_file = await db.get(ChatFileModel, UUID(fid))
    if chat_file.file_type == "image":
        file_data = await storage.load(chat_file.storage_path)
        image_parts.append(
            BinaryContent(data=file_data, media_type=chat_file.mime_type)
        )
    elif chat_file.parsed_content:
        file_context_parts.append(
            f"\n---\nAttached file: {chat_file.filename}"
            f"\n```\n{chat_file.parsed_content}\n```"
        )
```

`BinaryContent` von [Pydantic AI](https://ai.pydantic.dev/) umhüllt Rohbytes mit einem MIME-Typ und lässt das Framework den multimodalen API-Aufruf übernehmen. Das LLM sieht das Bild buchstäblich.
