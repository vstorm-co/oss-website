## Das Problem mit unsichtbaren Agenten

Wenn ein KI-Agent eine Aufgabe ausführt, passiert zwischen deinem Prompt und der Antwort viel. Das Modell denkt. Es ruft Werkzeuge auf. Es liest Dateien, führt Suchen durch, aktualisiert Aufgaben, delegiert an Subagenten. Jede Aktion verbraucht Tokens. Jede Antwort kostet Geld.

Ohne Observability fliegst du blind.

In v0.3.5 haben wir das TUI überarbeitet, um alles sichtbar zu machen.

## Was sich geändert hat

### Token-Nutzung pro Runde

Jede Assistenten-Antwort zeigt jetzt eine Token-Aufschlüsselung direkt darunter:

```
in:2.1K · out:412 · total:2.5K · reqs:3
```

- `in`, Tokens im Input (Prompt + Kontext)
- `out`, Tokens in der Antwort
- `total`, Summe für diese Runde
- `reqs`, Anzahl der API-Aufrufe in dieser Runde

### Kumulative Kosten in der Kopfzeile

Die obere Leiste zeigt laufende Gesamtmengen nach jeder Antwort:

```
pydantic-deep  in:45K out:3K · $0.12
```

Wird live aktualisiert.

### Denken live gestreamt, dann zusammengeklappt

Wenn ein Modell vor der Antwort überlegt (Claude Extended Thinking, o3-Reasoning), erscheint dieser Inhalt im TUI als gedimmter Text, live gestreamt. Nach Abschluss der Reasoning-Phase klappt er zu einer einzigen Zusammenfassungszeile zusammen.

### Seitenpanel standardmäßig sichtbar

Bei einem Terminal mit ≥100 Zeichen Breite öffnet sich das Seitenpanel beim Start. Es zeigt verfügbare Subagenten vor Beginn jeder Arbeit:

```
Subagents:
• planner (idle)
• research (idle)
```

### Alle Tool-Aufrufe sichtbar

Todo-Werkzeuge (`read_todos`, `write_todos`, `add_todo`, `update_todo_status`, `remove_todo`) waren zuvor versteckt. Jetzt ist jede Aktion des Agenten sichtbar.

### Sitzung auch bei Absturz gespeichert

`_save_session()` befindet sich jetzt in einem `finally`-Block. Was auch immer passiert, Absturz, Ausnahme, Tastaturunterbrechung, `messages.json` wird auf die Festplatte geschrieben.

### Subagenten-Logs: 20K Zeichen (war 2K)

`tool_log.jsonl` speichert jetzt bis zu 20K Zeichen für Subagenten-Aufgabenergebnisse. Wichtig für `/improve`, die Pipeline, die Erkenntnisse aus Sitzungen extrahiert.

## Das vollständige TUI-Layout

```
┌─────────────────────────────────┬──────────────────┐
│ pydantic-deep  in:45K out:3K · $0.12               │
├─────────────────────────────────┼──────────────────┤
│ [thinking... gedimmt]           │ Subagents:       │
│ [auf Zusammenfassung reduziert] │ • planner (idle) │
│                                 │ • research (idle)│
│ Agenten-Antwort hier...         │                  │
│ in:2.1K · out:412 · $0.04      │                  │
└─────────────────────────────────┴──────────────────┘
```

## Wichtigste Erkenntnisse

- Token-Aufschlüsselung pro Runde zeigt genau, wo Tokens verbraucht werden
- Kopfzeilen-Kosten werden live aktualisiert, sitzungsweit
- Denkinhalte werden als gedimmter Text gestreamt, nach Abschluss zusammengeklappt
- Seitenpanel zeigt Subagenten-Status, reagiert auf Terminal-Größe
- Alle Tool-Aufrufe sichtbar, einschließlich Todo-Verwaltungswerkzeuge
- Sitzung bei Absturz über `finally`-Block gespeichert
- Subagenten-Logs mit 20K Zeichen für `/improve`
