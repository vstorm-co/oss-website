Estuvimos en silencio durante dos semanas. Sin tweets, sin posts, sin anuncios "emocionantes".

Estábamos construyendo.

[pydantic-deep](https://github.com/vstorm-co/pydantic-deep) es nuestro agente de codificación de IA de código abierto — una alternativa Python-first a Claude Code y herramientas similares, construida sobre Pydantic AI. Entregamos características reales de nuestros deployments de producción: 30+ sistemas de agentes de IA en Vstorm, casos límite reales, fallos reales, correcciones reales.

En dos semanas, lanzamos v0.3.4 a v0.3.8. Cinco versiones. Esto es lo que cada una añadió.

## v0.3.4 — TUI como interfaz predeterminada + Rediseño de /improve

El mayor cambio de calidad de vida: ejecutar `pydantic-deep` ahora lanza el TUI por defecto. Sin más flags, sin más subcomandos.

Pero la parte más interesante es `/improve`. Rediseñamos todo el pipeline. En lugar de resumir sesiones de manera imprecisa, ahora extrae dos tipos distintos de insights:

- **`UserFactInsight`** — cosas que el agente aprendió sobre el usuario (preferencias, patrones, contexto)
- **`AgentLearningInsight`** — cosas que el agente aprendió sobre sí mismo (qué funcionó, qué falló)

Ambos alimentan a `MEMORY.md` como objetivo principal. Los rastros de herramientas sin procesar ahora se incluyen en la síntesis (arquitectura Meta-Harness: los datos crudos siempre superan a los resúmenes — los modelos razonan mejor con datos reales que con interpretaciones comprimidas).

Otras correcciones en v0.3.4: comando `/config` en TUI, registro de depuración por sesión en `.pydantic-deep/logs/`, corregido el crash de `/improve` con `{}` en prompts, fallos silenciosos ahora reportan `failed_sessions`, carga de claves API corregida, discordancia de ruta MEMORY.md corregida.

## v0.3.5 — Automatización de navegador, sandbox Docker, Headless Runner

Tres características de infraestructura aterrizaron juntas en v0.3.5.

**Automatización de navegador mediante `BrowserCapability`**: respaldada por Playwright, 9 herramientas:

```python
from pydantic_deep.capabilities import BrowserCapability

agent = create_deep_agent(
    model="anthropic:claude-opus-4-6",
    capabilities=[BrowserCapability()],
)
# El agente ahora puede: navigate, click, type_text, get_text,
# screenshot, scroll, go_back, go_forward, execute_js
```

Lista de dominios permitidos, auto-screenshot en navegación, ciclo de vida del navegador gestionado por `wrap_run`.

**Sandbox Docker (`--sandbox docker`)**: las operaciones de archivos y comandos de shell se ejecutan dentro de un contenedor. `/workspace` está montado, la limpieza es automática.

```bash
pydantic-deep --sandbox docker
```

**Workspaces con nombre**: entornos Docker persistentes que sobreviven entre sesiones, compartibles entre hilos.

```bash
pydantic-deep --workspace my-project  # reutilizar el mismo contenedor
pydantic-deep sandbox list            # ver workspaces en ejecución
pydantic-deep sandbox stop my-project
```

**Headless runner (`pydantic-deep run`)**: para benchmarks, CI/CD, automatización con scripts. Sin TUI, sin interacción.

También en v0.3.5: `DEFAULT_USAGE_LIMITS` elimina el límite predeterminado de 50 solicitudes de Pydantic AI. Visualización de tokens por turno (`in:X · out:Y · total:Z · reqs:N`) en el encabezado del TUI.

## v0.3.6 — Instalación con un comando

La mejora práctica más importante para nuevos usuarios:

```bash
curl -fsSL https://raw.githubusercontent.com/vstorm-co/pydantic-deep/main/install.sh | bash
```

Sin configuración de virtualenv, sin pip, sin conflictos de versiones.

Dos características más incluidas:

- **`pydantic-deep update`** — comando de auto-actualización. Ejecútalo, obtén la última versión.
- **Notificaciones de actualización al inicio** — al lanzar, pydantic-deep verifica versiones más recientes (caché de 24h, timeout de 2s para no bloquear el inicio). Si hay una actualización, te lo dice.

## v0.3.8 — Tres capabilities autoconscientes

### StuckLoopDetection

```python
from pydantic_deep import create_deep_agent

agent = create_deep_agent(
    model="anthropic:claude-opus-4-6",
    stuck_loop_detection=True,  # predeterminado: True
)
```

Tres patrones detectados:
1. **Llamadas idénticas repetidas** — misma herramienta, mismos argumentos, N veces seguidas (umbral predeterminado: 3)
2. **Alternancia A-B-A-B** — herramienta A, herramienta B, herramienta A, herramienta B... atrapada en un bucle de dos llamadas
3. **Bucles no-op** — misma llamada, mismo resultado, sigue repitiéndose

Acción configurable: `warn` (activa `ModelRetry`) o `error` (lanza `StuckLoopError`). Aislamiento per-run mediante `for_run()`.

### LimitWarnerCapability

Inyecta advertencias directamente en la conversación como mensajes de usuario:

- Al **70% de uso del contexto**: `[URGENT] Context window at 70%. Start wrapping up current task.`
- Al **90% de uso del contexto**: `[CRITICAL] Context window at 90%. Auto-compression imminent.`

Habilitado automáticamente cuando `context_manager=True` (predeterminado).

### Búsqueda de historial BM25

`search_conversation_history` ahora usa puntuación BM25 en lugar de comparación de subcadenas naïve. Misma fórmula que Elasticsearch y Lucene. Cero dependencias — implementación Python pura.
