## Por que "Shields" y no "Middleware"?

Este paquete se llamaba `pydantic-ai-middleware`. Cuando Pydantic AI v1.71 entrego [capabilities, hooks y agent specs](/blog/pydantic-ai-capabilities-hooks-agent-specs/) nativos, la capa de middleware se volvio redundante â€” el framework hace composicion, ordenamiento de hooks y manejo de errores mejor que nosotros.

Asi que borramos todo. `MiddlewareAgent`, `MiddlewareChain`, `ParallelMiddleware`, el compilador de pipeline â€” miles de lineas eliminadas. Lo que quedo fue la parte que realmente importaba: **guardrails**. Escudos que protegen a tu agente de input malo, output malo y costos descontrolados.

El cambio de nombre a `pydantic-ai-shields` refleja ese enfoque. Ya no es middleware. Es una biblioteca de guardrails construida sobre la API nativa de capabilities de Pydantic AI.

```bash
pip install pydantic-ai-shields
```

## Como funciona

Cada shield es una capability de Pydantic AI. Las pasas al parametro `capabilities` de tu agente y se componen automaticamente:

```python
from pydantic_ai import Agent
from pydantic_ai_shields import (
    CostTracking,
    PromptInjection,
    PiiDetector,
    SecretRedaction,
)

agent = Agent(
    "openai:gpt-4.1",
    capabilities=[
        CostTracking(budget_limit_usd=1.00),
        PromptInjection(),
        PiiDetector(),
        SecretRedaction(),
    ],
)
```

Sin agente wrapper. Sin cadena personalizada. Solo capabilities que se conectan y funcionan.

## Las 5 capabilities de infraestructura

Manejan el lado operativo â€” seguimiento de costos, control de acceso a herramientas y validacion de entrada/salida.

### 1. CostTracking

Rastrea el uso de tokens y el costo en USD entre ejecuciones. Aplica limites de presupuesto.

```python
from pydantic_ai_shields import CostTracking

cost = CostTracking(budget_limit_usd=5.00)

agent = Agent("openai:gpt-4.1", capabilities=[cost])
result = await agent.run("Analiza este dataset")

print(cost.total_cost_usd)   # 0.0023
print(cost.total_tokens)     # 1847
```

Cuando se excede el presupuesto, el agente lanza `BudgetExceededError` antes de la siguiente solicitud al modelo.

### 2. ToolGuard

Bloquea herramientas especificas o requiere aprobacion antes de ejecutar.

```python
from pydantic_ai_shields import ToolGuard

guard = ToolGuard(
    blocked_tools=["delete_user", "drop_table"],
    require_approval=["send_email"],
    approval_fn=lambda tool_name, args: input(f"Allow {tool_name}? (y/n): ") == "y",
)
```

### 3. InputGuard

Ejecuta una verificacion en cada input del usuario antes de que llegue al modelo.

```python
from pydantic_ai_shields import InputGuard

def check_input(text: str) -> str | None:
    if len(text) > 10_000:
        return "Input too long â€” max 10,000 characters."
    return None  # None = permitir

agent = Agent("openai:gpt-4.1", capabilities=[InputGuard(guard_fn=check_input)])
```

### 4. OutputGuard

Mismo patron, pero para el output del modelo. Verifica o bloquea respuestas antes de que lleguen al usuario.

```python
from pydantic_ai_shields import OutputGuard

def check_output(text: str) -> str | None:
    if "CONFIDENTIAL" in text.upper():
        return "Response contained confidential information and was blocked."
    return None

agent = Agent("openai:gpt-4.1", capabilities=[OutputGuard(guard_fn=check_output)])
```

### 5. AsyncGuardrail

Ejecuta un guardrail concurrentemente con la llamada al LLM. Tres modos de timing:

- **`before`** â€” el guardrail se completa antes de devolver la respuesta del LLM
- **`race`** â€” el primero en terminar gana (el guardrail puede cancelar la llamada al LLM)
- **`background`** â€” el guardrail corre en segundo plano, registra resultados, no bloquea

```python
from pydantic_ai_shields import AsyncGuardrail

async def content_policy_check(text: str) -> str | None:
    result = await check_content_policy(text)
    return "Policy violation" if result.flagged else None

agent = Agent(
    "openai:gpt-4.1",
    capabilities=[AsyncGuardrail(guard_fn=content_policy_check, timing="before")],
)
```

## Los 5 Content Shields integrados

Shields pre-construidos que detectan patrones de contenido especificos. Sin configuracion necesaria. Cero dependencias externas â€” todo basado en regex y heuristicas.

### 6. PromptInjection

Detecta prompt injection y intentos de jailbreak en 6 categorias: override de instrucciones, manipulacion de rol, escape de contexto, ingenieria social, ataques de encoding e inyeccion indirecta.

```python
from pydantic_ai_shields import PromptInjection

agent = Agent("openai:gpt-4.1", capabilities=[PromptInjection()])
agent = Agent("openai:gpt-4.1", capabilities=[PromptInjection(sensitivity="high")])
```

### 7. PiiDetector

Detecta informacion personal identificable: emails, telefonos, SSN, tarjetas de credito y direcciones IP.

```python
from pydantic_ai_shields import PiiDetector

agent = Agent("openai:gpt-4.1", capabilities=[PiiDetector(action="block")])
agent = Agent("openai:gpt-4.1", capabilities=[PiiDetector(action="log")])
```

### 8. SecretRedaction

Bloquea output del modelo que contiene claves API, tokens o credenciales. Patrones para OpenAI, Anthropic, AWS, GitHub, Slack, JWT y claves privadas.

```python
from pydantic_ai_shields import SecretRedaction

agent = Agent("openai:gpt-4.1", capabilities=[SecretRedaction()])
```

Este shield nos salvo en produccion. Un modelo con retrieval sobre documentos internos devolvio una respuesta que incluia una clave de acceso AWS de un archivo de configuracion en la base de conocimiento. SecretRedaction la intercepto antes de que llegara al usuario.

### 9. BlockedKeywords

Bloquea palabras, frases o patrones regex especificos.

```python
from pydantic_ai_shields import BlockedKeywords

agent = Agent(
    "openai:gpt-4.1",
    capabilities=[
        BlockedKeywords(
            keywords=["competitor_name", "internal_project"],
            case_sensitive=False,
            whole_word=True,
        )
    ],
)
```

### 10. NoRefusals

Detecta cuando el modelo se niega a responder y bloquea la negativa. 10 patrones integrados con soporte para negativas parciales.

```python
from pydantic_ai_shields import NoRefusals

agent = Agent("openai:gpt-4.1", capabilities=[NoRefusals()])
```

## Composicion de multiples shields

El verdadero poder esta en la composicion. Apila tantos shields como necesites â€” se componen automaticamente:

```python
from pydantic_ai import Agent
from pydantic_ai_shields import (
    CostTracking, PromptInjection, PiiDetector,
    SecretRedaction, InputGuard, OutputGuard, NoRefusals,
)

agent = Agent(
    "openai:gpt-4.1",
    capabilities=[
        CostTracking(budget_limit_usd=2.00),
        PromptInjection(sensitivity="medium"),
        PiiDetector(action="block"),
        SecretRedaction(),
        NoRefusals(),
        InputGuard(guard_fn=my_custom_input_check),
        OutputGuard(guard_fn=my_custom_output_check),
    ],
)
```
