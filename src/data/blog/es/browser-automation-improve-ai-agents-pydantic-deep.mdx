> **TL;DR:** pydantic-deep v0.3.5 trae `BrowserCapability`, 9 herramientas Playwright (navigate, click, type_text, get_text, screenshot, scroll, go_back, go_forward, execute_js) con allowlist de dominios, diseño single-tab e intercepción automática de popups. v0.3.4 rediseña `/improve`: extrae `UserFactInsight` y `AgentLearningInsight` de trazas de herramientas sin procesar (no resúmenes) a MEMORY.md. Hallazgo clave: las trazas sin procesar superan significativamente a los resúmenes para producir aprendizajes accionables.

## Parte 1: BrowserCapability

### Instalación

```bash
pip install 'pydantic-deep[browser]'
playwright install chromium
```

### Las 9 Herramientas

```python
from pydantic_deep.capabilities import BrowserCapability

agent = create_deep_agent(
    model="anthropic:claude-opus-4-6",
    extra_capabilities=[BrowserCapability(
        allowed_domains=["github.com", "docs.python.org"],
        auto_screenshot=True,
        headless=False,  # ventana visible por defecto
    )]
)
```

| Herramienta                 | Qué hace                                               |
| --------------------------- | ------------------------------------------------------ |
| `navigate(url)`             | Navega a URL, devuelve contenido + screenshot opcional |
| `click(selector)`           | Hace clic en elemento por selector CSS                 |
| `type_text(selector, text)` | Escribe en campo de entrada                            |
| `get_text(selector)`        | Extrae texto de elemento                               |
| `screenshot()`              | Captura estado actual de la página                     |
| `scroll(direction, amount)` | Desplaza página arriba/abajo                           |
| `go_back()`                 | Retroceder en navegador                                |
| `go_forward()`              | Avanzar en navegador                                   |
| `execute_js(script)`        | Ejecuta JavaScript arbitrario                          |

Cada herramienta devuelve un `BrowseResult`: `url`, `title`, `content`, `screenshot`, `error`.

### Diseño de Seguridad

**Single-tab.** Una pestaña a la vez, decisión deliberada, no una limitación. Múltiples pestañas requerirían que el agente rastree el contexto de pestañas.

**Allowlist de dominios.** El agente solo puede navegar a dominios explícitamente permitidos. Los intentos de navegar a otro lugar devuelven un error.

**Intercepción automática de popups.** Banners de cookies, diálogos de inicio de sesión, avisos de suscripción, interceptados automáticamente.

**Truncamiento de contenido.** El contenido de la página se trunca antes de pasarse al modelo.


**Flags CLI:**

```bash
pydantic-deep tui --browser                      # activar navegador (desactivado por defecto)
pydantic-deep tui --browser --browser-headless   # sin ventana (para CI)
pydantic-deep tui --browser --browser-headed     # ventana visible (predeterminado)

# Combinar con Docker sandbox
pydantic-deep run "research competitors on GitHub" \
  --browser \
  --sandbox docker
```

## Parte 2: /improve, El Bucle de Automejora

### Qué hace /improve

Después de una sesión del agente, ejecuta `/improve` en la TUI. Analiza la sesión completa y extrae dos categorías de insights:

**`UserFactInsight`**, lo que el agente aprendió sobre ti y tus preferencias.

**`AgentLearningInsight`**, patrones y estrategias que el agente descubrió.

Ambos tipos escriben en MEMORY.md, cargado al inicio de cada sesión posterior.

### El Hallazgo Meta-Harness: Trazas Sin Procesar >> Resúmenes

Comparamos dos entradas para el paso de síntesis:

1. Resúmenes legibles por humanos de lo que sucedió
2. Trazas de herramientas sin procesar, la secuencia real de llamadas a herramientas y respuestas

Las trazas sin procesar funcionaron significativamente mejor. Los resúmenes pierden los patrones de fallo/recuperación que contienen la señal de aprendizaje real. `tool_log.jsonl` (escrito por sesión) captura la traza completa de herramientas. `/improve` lo lee directamente.
