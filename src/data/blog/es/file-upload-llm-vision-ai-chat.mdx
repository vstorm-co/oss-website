> **TL;DR:** La version 0.2.2 del template full-stack AI agent agrega subida de archivos con soporte de LLM Vision. Las imagenes (PNG, JPG, WebP) se envian como `BinaryContent` directamente a la API de LLM Vision — el modelo las ve. Los PDFs se parsean del lado del servidor con PyMuPDF (bloques de texto + tablas convertidas a Markdown). Los archivos DOCX se parsean con python-docx. Todo el contenido parseado se inyecta en el contexto del agente. El flujo de subida usa REST para el manejo de archivos (validacion, parseo, almacenamiento) y WebSocket para mensajes de chat (referenciando file IDs). Un modelo de base de datos `ChatFile` rastrea el propietario, tipo MIME, ruta de almacenamiento y contenido parseado cacheado. El parser de PDF es configurable via variable de entorno `CHAT_PDF_PARSER` (PyMuPDF, LlamaParse o LiteParse). Funciona con los 5 frameworks de IA soportados.

## La arquitectura: tres rutas para tres tipos de archivo

Cuando un usuario sube un archivo en el chat, pueden pasar tres cosas dependiendo del tipo de archivo:

```
Usuario sube archivo
  │
  ├── Imagen (png/jpg/gif/webp)
  │     → Almacenar en media/
  │     → Cargar como BinaryContent
  │     → Enviar a la API de LLM Vision
  │     → El agente "ve" la imagen
  │
  ├── PDF
  │     → Almacenar en media/
  │     → Parsear con PyMuPDF (texto + tablas)
  │     → Inyectar texto parseado en el prompt del agente
  │     → El agente lee el contenido del documento
  │
  └── DOCX
        → Almacenar en media/
        → Parsear con python-docx
        → Inyectar texto parseado en el prompt del agente
        → El agente lee el contenido del documento
```

La decision de diseno clave: **imagenes y documentos toman caminos diferentes**. Las imagenes necesitan capacidades multimodales del LLM (Vision API). Los documentos necesitan extraccion de texto del lado del servidor. Ambos terminan en el contexto del agente, pero a traves de mecanismos fundamentalmente diferentes.

## Paso 1: Endpoint de subida con validacion

```python
@router.post("/upload", response_model=FileUploadResponse,
             status_code=status.HTTP_201_CREATED)
async def upload_file(
    file: UploadFile = File(...),
    file_upload_svc: FileUploadSvc = None,
    current_user: CurrentUser = None,
) -> Any:
    data = await file.read()
    is_valid, error = file_upload_svc.validate_upload(
        file.content_type, len(data)
    )
    if not is_valid:
        raise HTTPException(status_code=400, detail=error)

    file_type = file_upload_svc.classify_file(
        file.content_type or "", file.filename or "unknown"
    )
    parsed_content = await file_upload_svc.parse_content(
        data, file_type, file.content_type or ""
    )

    storage = get_file_storage()
    storage_path = await storage.save(
        str(current_user.id), file.filename or "unknown", data
    )

    chat_file = await file_upload_svc.create_chat_file(
        user_id=current_user.id,
        filename=file.filename or "unknown",
        mime_type=file.content_type or "application/octet-stream",
        size=len(data),
        storage_path=storage_path,
        file_type=file_type,
        parsed_content=parsed_content,
    )
    return FileUploadResponse(
        id=chat_file.id, filename=chat_file.filename,
        mime_type=chat_file.mime_type, size=chat_file.size,
        file_type=chat_file.file_type,
    )
```

## Paso 2: Parseo de PDF con PyMuPDF

```python
@staticmethod
def _parse_pdf_content(data: bytes) -> str | None:
    import pymupdf
    doc = pymupdf.open(stream=data, filetype="pdf")
    texts = []
    for page in doc:
        blocks = page.get_text("blocks")
        for b in blocks:
            if b[6] == 0:
                text = b[4].strip()
                if text:
                    texts.append(text)
        tables = page.find_tables()
        if tables and tables.tables:
            for table in tables.tables:
                df = table.to_pandas()
                if not df.empty:
                    texts.append(df.to_markdown(index=False))
    doc.close()
    return "\n\n".join(texts) if texts else None
```

```python
from pydantic_ai.messages import BinaryContent

for fid in file_ids:
    chat_file = await db.get(ChatFileModel, UUID(fid))
    if chat_file.file_type == "image":
        file_data = await storage.load(chat_file.storage_path)
        image_parts.append(
            BinaryContent(data=file_data, media_type=chat_file.mime_type)
        )
    elif chat_file.parsed_content:
        file_context_parts.append(
            f"\n---\nAttached file: {chat_file.filename}"
            f"\n```\n{chat_file.parsed_content}\n```"
        )
```

`BinaryContent` de [Pydantic AI](https://ai.pydantic.dev/) envuelve bytes crudos con un tipo MIME y deja que el framework maneje la llamada a la API multimodal. El LLM literalmente ve la imagen.
