> **TL;DR:** Nasz open-source'owy szablon full-stack AI wspiera 5 frameworkow agentow (Pydantic AI, LangChain, LangGraph, CrewAI, DeepAgents) za jednym interfejsem. Warstwa agenta jest pluginem — streaming WebSocket, persystencja konwersacji, upload plikow, pipeline RAG, autoryzacja i panel admina to wspolna infrastruktura, ktora nie zmienia sie przy przelaczaniu frameworkow. Kazdy framework dostaje wlasny modul asystenta, a `agents/__init__.py` re-eksportuje odpowiednie klasy na podstawie wybranego frameworka. Handler WebSocket importuje z `app.agents` bez wiedzy o tym, ktora implementacja stoi za nim. Wybor frameworka odbywa sie podczas generowania projektu przez `--ai-framework pydantic_ai`.

"Ktorego frameworka AI powinienem uzyc?"

Zle pytanie. Prawidlowe pytanie brzmi: "Jak budowac, zeby framework nie mial znaczenia?"

Co kilka miesiecy ekosystem Python AI dostaje nowego pretendenta. LangChain dominowal w 2024. Pydantic AI zyskal powaznie na popularnosci w 2025. LangGraph, CrewAI i DeepAgents wyrobily sobie nisze. Kazdy ma kompromisy. Zaden nie jest uniwersalnie lepszy.

Zmeczylismy sie odbudowywaniem tej samej infrastruktury FastAPI + Next.js za kazdym razem, gdy klient chcial innego frameworka. Wiec zbudowalismy jeden szablon, ktory wspiera wszystkie piec — a framework staje sie pojedynczym wyborem konfiguracji.

Oto jak dziala architektura, z prawdziwym kodem z [open-source'owego szablonu](https://github.com/vstorm-co/full-stack-ai-agent-template).

## Zasada: warstwa agenta jako plugin

```
┌──────────────────────────────────────────┐
│                 Frontend                  │
│   Next.js 15 · React 19 · Tailwind v4    │
├──────────────────────────────────────────┤
│              Warstwa WebSocket            │
│   Streaming · Eventy · Obsluga plikow    │
├──────────────────────────────────────────┤
│           Warstwa agenta  ← WYMIENNA     │
│   Pydantic AI | LangChain | LangGraph    │
│   CrewAI | DeepAgents                     │
├──────────────────────────────────────────┤
│           Wspolne serwisy                 │
│   Auth · DB · RAG · Upload · Narzedzia   │
└──────────────────────────────────────────┘
```

Wszystko powyzej i ponizej warstwy agenta jest wspolne. Frontend nie wie, ktory framework dziala. Schemat bazy danych jest identyczny. Protokol WebSocket jest taki sam. [Pipeline RAG](/blog/full-rag-pipeline-4-vector-stores/), [upload plikow](/blog/file-upload-llm-vision-ai-chat/), autoryzacja — wszystko niezalezne od frameworka.

## Wybor frameworka

```bash
pip install fastapi-fullstack
fastapi-fullstack --ai-framework pydantic_ai  # lub langchain, langgraph, crewai, deepagents
```

Generator szablonow (Cookiecutter) uzywa tego wyboru, zeby dolaczac tylko odpowiedni modul agenta:

```python
# Pydantic AI:
from app.agents.assistant import AssistantAgent, Deps

# LangChain:
from app.agents.langchain_assistant import LangChainAssistant, AgentContext, AgentState

# LangGraph:
from app.agents.langgraph_assistant import LangGraphAssistant, AgentContext, AgentState

# CrewAI:
from app.agents.crewai_assistant import CrewAIAssistant, CrewConfig, CrewContext

# DeepAgents:
from app.agents.deepagents_assistant import DeepAgentsAssistant, DeepAgentsDeps
```

## Kiedy uzyc ktorego frameworka

| Framework | Najlepszy do | Kompromis |
|-----------|-------------|-----------|
| **Pydantic AI** | Typowane agenty, structured output, multimodal | Nowszy ekosystem |
| **LangChain** | Zespoly znajace LangChain, LangSmith | Ciezkie abstrakcje |
| **LangGraph** | Zlozony workflow, jawne maszyny stanow | Stroma krzywa uczenia |
| **CrewAI** | Wspolpraca wielu agentow, zadania rolowe | Overhead dla prostych przypadkow |
| **DeepAgents** | Middleware, human-in-the-loop | Najmniejszy ekosystem |

Szczera odpowiedz: dla wiekszosci projektow, wybierz ten, ktory Twoj zespol zna najlepiej. Infrastruktura wokol agenta ma wieksze znaczenie niz sam framework.

## Wyprobuj

```bash
pip install fastapi-fullstack
fastapi-fullstack
# Wybierz framework → produkcyjna aplikacja w 30 sekund
```
