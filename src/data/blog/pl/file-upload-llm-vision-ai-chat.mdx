> **TL;DR:** Wersja 0.2.2 szablonu full-stack AI agent dodaje upload plików ze wsparciem LLM Vision. Obrazy (PNG, JPG, WebP) są wysyłane jako `BinaryContent` bezpośrednio do API LLM Vision — model je widzi. PDF-y są parsowane po stronie serwera za pomocą PyMuPDF (bloki tekstu + tabele konwertowane do Markdown). Pliki DOCX parsowane python-docx. Cała sparsowana treść jest wstrzykiwana do kontekstu agenta. Flow uploadu wykorzystuje REST do obsługi plików (walidacja, parsowanie, zapis) i WebSocket do wiadomości czatu (referencja przez file IDs). Model `ChatFile` w bazie danych śledzi właściciela, typ MIME, ścieżkę zapisu i zcache'owaną sparsowaną treść. Parser PDF jest konfigurowalny przez zmienną `CHAT_PDF_PARSER` (PyMuPDF, LlamaParse lub LiteParse). Działa ze wszystkimi 5 wspieranymi frameworkami AI.

## Architektura: trzy ścieżki dla trzech typów plików

Kiedy użytkownik uploaduje plik w czacie, mogą się wydarzyć trzy rzeczy w zależności od typu pliku:

```
Użytkownik uploaduje plik
  │
  ├── Obraz (png/jpg/gif/webp)
  │     → Zapis w media/
  │     → Załadowanie jako BinaryContent
  │     → Wysłanie do API LLM Vision
  │     → Agent "widzi" obraz
  │
  ├── PDF
  │     → Zapis w media/
  │     → Parsowanie PyMuPDF (tekst + tabele)
  │     → Wstrzyknięcie sparsowanego tekstu do promptu agenta
  │     → Agent czyta treść dokumentu
  │
  └── DOCX
        → Zapis w media/
        → Parsowanie python-docx
        → Wstrzyknięcie sparsowanego tekstu do promptu agenta
        → Agent czyta treść dokumentu
```

Kluczowa decyzja projektowa: **obrazy i dokumenty idą różnymi ścieżkami**. Obrazy wymagają multimodalnych możliwości LLM (Vision API). Dokumenty wymagają ekstrakcji tekstu po stronie serwera. Oba trafiają do kontekstu agenta, ale przez fundamentalnie różne mechanizmy.

## Krok 1: Endpoint uploadu z walidacją

```python
@router.post("/upload", response_model=FileUploadResponse,
             status_code=status.HTTP_201_CREATED)
async def upload_file(
    file: UploadFile = File(...),
    file_upload_svc: FileUploadSvc = None,
    current_user: CurrentUser = None,
) -> Any:
    """Upload a file for use in chat."""
    data = await file.read()

    # Validate type and size
    is_valid, error = file_upload_svc.validate_upload(
        file.content_type, len(data)
    )
    if not is_valid:
        raise HTTPException(status_code=400, detail=error)

    # Classify: image, pdf, docx, or text
    file_type = file_upload_svc.classify_file(
        file.content_type or "", file.filename or "unknown"
    )

    # Parse content for non-image files
    parsed_content = await file_upload_svc.parse_content(
        data, file_type, file.content_type or ""
    )

    # Store file on disk
    storage = get_file_storage()
    storage_path = await storage.save(
        str(current_user.id), file.filename or "unknown", data
    )

    # Create database record
    chat_file = await file_upload_svc.create_chat_file(
        user_id=current_user.id,
        filename=file.filename or "unknown",
        mime_type=file.content_type or "application/octet-stream",
        size=len(data),
        storage_path=storage_path,
        file_type=file_type,
        parsed_content=parsed_content,
    )

    return FileUploadResponse(
        id=chat_file.id, filename=chat_file.filename,
        mime_type=chat_file.mime_type, size=chat_file.size,
        file_type=chat_file.file_type,
    )
```

## Krok 2: Parsowanie PDF za pomocą PyMuPDF

```python
@staticmethod
def _parse_pdf_content(data: bytes) -> str | None:
    """Extract text from PDF using PyMuPDF."""
    import pymupdf

    doc = pymupdf.open(stream=data, filetype="pdf")
    texts = []
    for page in doc:
        blocks = page.get_text("blocks")
        for b in blocks:
            if b[6] == 0:  # text block
                text = b[4].strip()
                if text:
                    texts.append(text)

        # Extract tables as markdown
        tables = page.find_tables()
        if tables and tables.tables:
            for table in tables.tables:
                df = table.to_pandas()
                if not df.empty:
                    texts.append(df.to_markdown(index=False))
    doc.close()
    return "\n\n".join(texts) if texts else None
```

Dwa kluczowe elementy:

1. **Tabele stają się Markdownem.** `page.find_tables()` + `to_pandas()` + `to_markdown()` zachowuje strukturę tabel, którą zwykłe `get_text()` spłaszczyłoby do nieczytelnego tekstu.

2. **Ekstrakcja na poziomie bloków.** `get_text("blocks")` zamiast `get_text("text")` daje nam dane pozycyjne. Filtrujemy po typie bloku `b[6] == 0`, żeby pominąć bloki obrazów.

## Krok 3: Obrazy przez LLM Vision

````python
from pydantic_ai.messages import BinaryContent

for fid in file_ids:
    chat_file = await db.get(ChatFileModel, UUID(fid))

    if chat_file.file_type == "image":
        file_data = await storage.load(chat_file.storage_path)
        image_parts.append(
            BinaryContent(
                data=file_data,
                media_type=chat_file.mime_type
            )
        )
    elif chat_file.parsed_content:
        file_context_parts.append(
            f"\n---\nAttached file: {chat_file.filename}"
            f"\n```\n{chat_file.parsed_content}\n```"
        )
````

`BinaryContent` z [Pydantic AI](https://ai.pydantic.dev/) opakowuje surowe bajty z typem MIME i pozwala frameworkowi obsłużyć multimodalne wywołanie API. LLM dosłownie widzi obraz — może opisywać wykresy, czytać tekst ze zrzutów ekranu, analizować diagramy.
