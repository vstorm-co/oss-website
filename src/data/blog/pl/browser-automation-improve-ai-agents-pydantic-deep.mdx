> **TL;DR:** pydantic-deep v0.3.5 wprowadza `BrowserCapability`, 9 narzędzi Playwright (navigate, click, type_text, get_text, screenshot, scroll, go_back, go_forward, execute_js) z allowlistą domen, projektem single-tab i automatycznym przechwytywaniem popupów. v0.3.4 przeprojektowuje `/improve`: wyciąga `UserFactInsight` i `AgentLearningInsight` z surowych śladów narzędzi (nie podsumowań) do MEMORY.md. Kluczowe odkrycie: surowe ślady znacznie przewyższają podsumowania w produkowaniu użytecznych wniosków.

W tym tygodniu opisałem [StuckLoopDetection](/blog/stuck-loop-detection-ai-agents-pydantic-deep/) (poniedziałek), [ślepotę na okno kontekstu](/blog/context-window-limit-warner-bm25-pydantic-deep/) (wtorek), instalację jedną komendą (środa) i [Docker sandbox](/blog/ai-agents-docker-sandbox-ci-cd-pydantic-deep/) (czwartek). Dziś: dwie funkcje, które dopełniają obraz.

pydantic-deep to modularne środowisko uruchomieniowe agentów dla Pythona. Agent, który nie widzi sieci, jest ograniczony do tego, co już wie i co jest w Twojej bazie kodu. Agent, który nie uczy się ze swoich sesji, zaczyna od zera przy każdym uruchomieniu. Oba ograniczenia są do rozwiązania, i w pydantic-deep v0.3.5 i v0.3.4 je rozwiązaliśmy.

## Część 1: BrowserCapability

### Instalacja

```bash
pip install 'pydantic-deep[browser]'
playwright install chromium
```

### 9 narzędzi

```python
from pydantic_deep.capabilities import BrowserCapability

agent = create_deep_agent(
    model="anthropic:claude-opus-4-6",
    extra_capabilities=[BrowserCapability(
        allowed_domains=["github.com", "docs.python.org"],
        auto_screenshot=True,
        headless=False,  # widoczne okno domyślnie
    )]
)
```

| Narzędzie | Co robi |
|-----------|---------|
| `navigate(url)` | Nawigacja do URL, zwraca zawartość + opcjonalny screenshot |
| `click(selector)` | Kliknięcie elementu przez selektor CSS |
| `type_text(selector, text)` | Wpisanie tekstu w pole |
| `get_text(selector)` | Wyciągnięcie tekstu z elementu |
| `screenshot()` | Zapis stanu strony |
| `scroll(direction, amount)` | Scroll w górę/dół |
| `go_back()` | Cofnij w przeglądarce |
| `go_forward()` | Do przodu w przeglądarce |
| `execute_js(script)` | Wykonanie JavaScript |

Każde narzędzie zwraca `BrowseResult`: `url`, `title`, `content`, `screenshot`, `error`.

### Bezpieczeństwo

**Single-tab.** Jedna zakładka na raz, celowy wybór, nie ograniczenie. Wiele zakładek wymagałoby od agenta śledzenia kontekstu zakładek, co dodaje złożoność stanu i punkty awarii.

**Allowlista domen.** Agent może nawigować tylko do jawnie dozwolonych domen. Próby nawigacji gdzie indziej zwracają błąd.

**Automatyczne przechwytywanie popupów.** Bannery cookies, dialogi logowania, monity subskrypcji, przechwytywane automatycznie.

**Obcinanie treści.** Zawartość strony jest obcinana przed przekazaniem do modelu, zapobiegając przepełnieniu kontekstu.

### Cykl życia przeglądarki

Przeglądarka nie startuje leniwie przy pierwszym wywołaniu `navigate`, startuje przed uruchomieniem agenta i jest gwarantowana do zamknięcia po zakończeniu:

```python
# Cykl życia przez wzorzec wrap_run:
# Chromium startuje → agent działa → Chromium zamyka się
# Gwarantowane przy: sukcesie / wyjątku / anulowaniu
```

Żadnych osieroconych procesów przeglądarki niezależnie od tego, jak uruchomienie się kończy.

**Flagi CLI:**

```bash
pydantic-deep tui --browser                      # włącz przeglądarkę (domyślnie wyłączona)
pydantic-deep tui --browser --browser-headless   # bez okna (dla CI)
pydantic-deep tui --browser --browser-headed     # widoczne okno (domyślnie)

# Połącz z Docker sandbox
pydantic-deep run "research competitors on GitHub" \
  --browser \
  --sandbox docker
```

## Część 2: /improve, pętla samodoskonalenia

### Co robi /improve

Po sesji agenta uruchom `/improve` w TUI. Analizuje pełną sesję i wyciąga dwie kategorie wniosków:

**`UserFactInsight`**, co agent nauczył się o Tobie i Twoich preferencjach:
- Preferencje stylu kodu
- Preferowane formaty wyjścia
- Terminologia dziedzinowa, której używasz
- Wzorce w typach zadań, które dajesz

**`AgentLearningInsight`**, wzorce i strategie, które agent odkrył:
- Podejścia, które działały dla Twojej bazy kodu
- Efektywne sekwencje wywołań narzędzi
- Napotkane tryby awaryjne i jak były naprawiane
- Ograniczenia lub osobliwości Twojego projektu

Oba typy zapisują do MEMORY.md, ładowanego na początku każdej kolejnej sesji.

### Odkrycie Meta-Harness: surowe ślady >> podsumowania

Porównaliśmy dwa wejścia dla kroku syntezy:
1. Czytelne dla człowieka podsumowania tego, co się wydarzyło
2. Surowe ślady narzędzi, faktyczna sekwencja wywołań narzędzi i odpowiedzi

Surowe ślady działały znacznie lepiej. Podsumowania tracą wzorce awarii/naprawy, które zawierają faktyczny sygnał uczenia. "Agent próbował X, nie powiodło się, potem próbował Y, co zadziałało" staje się "agent rozwiązał problem" w podsumowaniu, ale uczenie się jest w awarii/naprawie.

Plik `tool_log.jsonl` (pisany per sesja) przechwytuje pełny ślad narzędzi. `/improve` czyta go bezpośrednio.
